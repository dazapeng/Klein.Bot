什么是 MCP？

Model Context Protocol (MCP) 是一个开放协议，允许 AI 应用与外部工具和数据源进行标准化交互。通过 MCP，KleinBot 可以：

  • 🔗 连接本地文件和数据库
  • 🌐 访问 Web API 和在线服务
  • 📊 集成数据分析工具
  • 🤖 调用其他 AI 模型和工具
  • 📝 管理项目文档和知识库

常见问题

找不到答案？加入我们的 Discord 社区

KleinBot 是免费的吗？ +
KleinBot 本身是开源免费的。您只需要支付使用的 AI 模型 API 费用（如 Anthropic Claude、OpenAI 等）。也支持使用免费的本地模型（通过 Ollama）。
KleinBot 支持哪些编辑器？ +
VS Code 扩展版支持 VS Code、Cursor、VSCodium、Windsurf 等基于 VS Code 的编辑器。CLI 版本可以在任何终端中使用，不受编辑器限制。
如何保证代码安全性？ +
KleinBot 的所有操作都在本地进行，代码不会上传到任何第三方服务器。您使用的 AI 模型 API 调用遵循对应服务商的数据处理政策。检查点功能让您可以随时回退到之前的状态。
Plan & Act 模式是什么？ +
Plan 模式下，KleinBot 会先分析任务并制定详细的执行计划，供您审核。Act 模式下，KleinBot 会按照计划自动执行操作。这种双模式设计让 AI 操作更可控、更透明，避免意外修改。
如何使用本地模型（如 Ollama）？ +
首先安装 Ollama 并下载模型（如 llama3.3），然后在 KleinBot 设置中选择 Ollama 作为模型提供商，配置本地 API 地址（默认 http://localhost:11434）即可使用。
KleinBot 和 Cline 有什么区别？ +
KleinBot 提供了独特的 Plan & Act 双模式、检查点回退系统、以及更丰富的模型支持（30+ vs 仅 Anthropic）。同时 Klein.Bot 提供 CLI 版本，更适合自动化和 CI/CD 场景。两者都是优秀的开源项目，可以根据需求选择。
如何参与 KleinBot 开发？ +
欢迎在 GitHub 上提交 Issue 和 Pull Request！我们欢迎各种形式的贡献：功能建议、Bug 报告、代码贡献、文档改进等。请查看 GitHub 仓库的 CONTRIBUTING.md 了解详情。
